Project 1 - Website

Building my website was an enlightening process that pushed me to think critically about both design and functionality. From the start, I wanted my site to be visually appealing, user-friendly, and accessible across different devices. This required careful planning and numerous decisions about layout, color schemes, typography, and navigation. I began by sketching wireframes to visualize how each page would look and how users would interact with the site. This step was crucial because it allowed me to identify the layout and style of my website before I wrote a single line of code. As I transitioned to development, I relied heavily on HTML for structuring content and CSS for styling. One of the key challenges I encountered was implementing a profile picture that was the perfect size. Initially, the image appeared either too large or too small depending on the screen, which disrupted the layout. I solved this by using CSS properties like max-width, height: auto, and flexbox alignment to ensure the image scaled correctly while staying centered. Another challenge was adding functional “Home” and “GitHub” buttons. I wanted them to be both visually appealing and easy to use. After experimenting with different styles, I used CSS hover effects and padding adjustments to make the buttons responsive and interactive, linking the GitHub button to my repository. These challenges taught me the importance of attention to detail in design and how small details in HTML and CSS can significantly improve the user experience. I also incorporated Git and GitHub into my workflow, which improved my project management skills. Committing changes frequently and using branches allowed me to experiment with new features without risking the stability of the main site. For instance, I created a separate branch to try out a new CSS animation effect, when it didn't work as intended, I was able to revert without affecting my progress. Throughout the project, I gained a deeper understanding of web development tools like Visual Studio Code, which streamlined my coding through extensions and live previews. I also realized the value of debugging, as small mistakes in syntax could break an entire layout. Looking forward, I want to enhance my site's interactivity by learning more in computing tec over the next terms. I also plan to improve accessibility, ensuring the site is fully navigable for all users. Overall, this project strengthened my technical skills, problem-solving abilities, and understanding of user-centered design.
